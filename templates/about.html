About this project

This project was created by a group of students for a National Language Processing class in an online Masters of Data Science program offered by Indiana University in the fall of 2023. Team members:

  • Aimee Flynn
  • Barza Fayazi-Azad
  • Paul Miller



Objectives

Create a music recommendation engine that users provide criteria to, which returns a list of songs that match the criteria that the user provided to them. The main problem we are looking to solve is the issue where a user is not being given music recommendations that 100% mean their tastes and preferences. With our recommendation engine, a user should be able to provide a song title, artist, or even genre that they like, and our recommendation engine will return a list of songs that match what they are looking for given the criteria provided.



Data

Data will be sourced from an existing Kaggle dataset which contains information on the top 2000 songs.
Source: Spotify dataset on Kaggle

After extracting from a .zip file, the data is contained in a .csv file containing 1994 records.

Each row has the following data fields: 'Index', 'Title', 'Artist', 'Top Genre', 'Year', 'Beats Per Minute (BPM)', 'Energy', 'Danceability', 'Loudness (dB)', 'Liveness', 'Valence', 'Length (Duration)', 'Acousticness', 'Speechiness', 'Popularity'

Data for this project will consist of only the Title, Artist and Top Genre fields. All other columns contain numeric characteristics for each song which are not relevant.
